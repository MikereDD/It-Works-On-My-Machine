# Sandalphon v3 Series

The v3 series introduced dashboard and monitoring capabilities, expanding Sandalphon beyond Telegram into a web-accessible music management platform. This release cycle focused on visibility, statistics, library browsing, and future dashboard infrastructure.

---

## v3.0 — Dashboard Foundation

### Overview

v3.0 established the foundation for future dashboard and monitoring functionality.

### New Commands

```text
/status
/stats
/exportlibrary
```

### Features

#### Status Reporting

Provides real-time information about:

- Bot status
- Queue state
- Cache status
- Runtime information

#### Statistics Reporting

Exposes dashboard-ready metrics including:

- Cache usage
- Library size
- Playlist statistics
- General bot activity

#### Library Export

Supports exporting library data as JSON snapshots for external tools and future integrations.

#### Local Bot API Visibility

Reports Local Bot API availability directly through Telegram.

### Result

Created the foundation for future web dashboard integration while improving visibility into bot operations.

---

## v3.1 — Flask Dashboard

### Overview

v3.1 introduced the first web-based dashboard for Sandalphon.

### Dashboard Pages

```text
/
/library
/playlists
/failed
```

### JSON API Endpoints

```text
/stats.json
/library.json
/playlists.json
/failed.json
```

### Features

#### Read-Only Dashboard

Provides browser-based access to:

- Library contents
- Playlist history
- Failed queue entries
- System statistics

#### Searchable Library

Users can browse and search their music collection through a web interface.

#### Lightweight HTTP APIs

Cache and library data are exposed through simple JSON endpoints.

#### Independent Architecture

The dashboard is designed to evolve separately from the Telegram bot runtime.

### Result

Sandalphon gained its first web interface and external API layer.

## v3.1.2 — Dashboard Stability Fixes

### Overview

v3.1.2 focused on stabilizing dashboard startup and runtime behavior.

### Fixes

#### Flask Initialization Order

Moved Flask application initialization before route registration.

#### Runtime Stability

Resolved:

```text
NameError: name 'app' is not defined
```

which could occur during startup.

#### Platform Validation

Confirmed reliable operation on:

- Raspberry Pi
- Linux systems
- Standalone dashboard deployments

#### Dashboard Binding

Verified dashboard operation on:

```text
http://<host>:8181
```

### Result

Improved startup reliability and stabilized standalone dashboard deployments.

---

## v3.1.3 — Legacy Library Compatibility

### Overview

v3.1.3 improved compatibility with older library and cache entries.

### Features

#### Fallback Metadata Parsing

Adds support for legacy entries containing only display information.

#### Automatic Artist/Title Detection

Can derive metadata from:

```text
Artist - Title
```

and

```text
filename.ext
```

fallback formats.

#### Improved Library Rendering

Restores readable library listings without requiring:

- Library rebuilds
- Cache regeneration
- Metadata reprocessing

#### Metadata Resilience

Improves handling of:

- Missing metadata
- Incomplete cache entries
- Older library formats

### Result

Improved dashboard compatibility with historical libraries and reduced maintenance requirements.
