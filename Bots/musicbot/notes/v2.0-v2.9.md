# Sandalphon v2 Series

The v2 series transformed Sandalphon from a simple music downloader into a full-featured personal music platform. Major additions included library management, playlist ingestion, intelligent searching, artwork caching, failure recovery, and long-term playlist maintenance.

---

## v2.0 — Library Mode

### Overview

v2.0 introduced Sandalphon's library system, transforming the bot into a personal music platform rather than a simple download tool.

### New Features

#### Searchable Music Library

Downloaded tracks are automatically stored in a searchable local index.

New commands:

```text
/library
/find <query>
/play <query>
/clearlibrary
```

#### Instant Playback

Users can play previously downloaded tracks directly from the library without requiring a new download.

#### Library Management

The library index can be rebuilt or cleared independently of cached media files.

### Result

Sandalphon evolved from a downloader into a personal music system with persistent track storage and retrieval.

## v2.1 — Live Administration

### Overview

v2.1 introduced Telegram-based administration controls.

### New Features

#### Administrative Commands

```text
/reload
/restart
```

#### Live Reload Support

Configuration and runtime state can be reloaded without stopping the bot.

#### Remote Restart

Administrators can restart the bot directly from Telegram.

### Result

Improved maintenance and testing workflows, especially when running under tmux.

#### Live Status Messages

Multiple status updates were replaced with a single evolving message.

Visual progress indicators were added during:

## v2.3 — Smarter Library Search

### Overview

v2.3 introduced fuzzy matching and intelligent search ranking.

### New Features

#### Typo-Tolerant Search

```text
/find <query>
/play <query>
```

can now tolerate:

- Minor misspellings
- Partial artist names
- Partial song titles

#### Search Ranking

Added:

- Weighted search scoring
- Improved artist/title ranking
- Duplicate prevention

#### Play Tracking

Tracks local play counts for future ranking improvements.

### Result

## v2.4 — Artist & Album Browsing

### Overview

v2.4 expanded library browsing capabilities.

### New Commands

```text
/artists
/artist <artist>

/albums
/album <album or artist>
```

### Features

#### Artist Browsing

Browse all artists currently available in the library.

#### Album Browsing

#### Metadata Grouping

Tracks are automatically grouped by:

- Artist
- Album

### Result

## v2.5 — Smart Artwork Cache

### Overview

v2.5 introduced intelligent album art caching.

### New Features

#### Artwork Cache

Album artwork is stored and reused when available.

#### Artwork Fallbacks

Cached artwork is used when fresh thumbnails cannot be retrieved.

#### Reporting Improvements

Artwork cache statistics are now included in cache reporting.

### Result

Improved consistency while reducing repeated thumbnail processing and downloads.

---

## v2.6 — Playlist Ingestion

### Overview

v2.6 introduced large-scale playlist importing.

### New Commands

```text
/playlist <playlist url>
/playlist <playlist url> --library-only

/playlists
```

### Features

#### Playlist Scanning

Supports scanning entire playlists before processing.

#### Duplicate Avoidance

Existing library tracks are skipped whenever possible.

#### Playlist Tracking

Stores:

- Playlist history
- Import statistics
- Processing results

### Result

Large playlist imports became practical and manageable.

---

## v2.7 — Smarter Queue Ordering

### Overview

v2.7 improved playlist processing efficiency.

### New Features

#### Priority Queueing

During playlist imports:

- Uncached tracks process first
- Cached tracks process later

#### Library-Only Ingestion

```text
--library-only
```

supports silent library population without Telegram uploads.

#### Background Workflows

Queue system expanded to better support long-running ingestion jobs.

### Result

Faster playlist imports and improved large-scale library building.

---

## v2.8 — Failure Recovery System

### Overview

v2.8 introduced persistent tracking of failed operations.

### New Commands

```text
/failed
/retryfailed
/clearfailed
```

### Features

Tracks:

- Failed downloads
- Queue failures
- Oversized files
- Processing errors

#### Automatic Cleanup

Successful retries are automatically removed from the failed queue.

### Result

Improved reliability during large playlist imports and library population tasks.

---

## v2.9 — Playlist Synchronization

### Overview

v2.9 introduced playlist synchronization and update tracking.

### New Command

```text
/syncplaylist <playlist url>
/syncplaylist <playlist url> --library-only
```

### Features

#### Playlist Fingerprinting

Tracks playlist entries using unique fingerprints.

#### Incremental Sync

Only new playlist entries are queued during synchronization.

#### History Tracking

Playlist history now records:

- Sync counts
- Update activity
- New additions

### Result

Playlists can now be maintained long-term without reprocessing existing content.
