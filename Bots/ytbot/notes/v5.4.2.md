i# 📝 v5.4.2 — Config-Driven Platform Support

> Control what the bot supports — without touching code.

---

## 🔥 Problem

Previously, supported platforms were **hardcoded**:

```text id="7b9p3a"
SUPPORTED_VIDEO_DOMAINS = (...)
```

This caused:

* Code edits required to add/remove platforms
* Risk of breaking validation logic
* No flexibility per environment

---

## 🧠 Root Cause

Platform support was:

* static
* global
* not configurable

---

## ✅ Fix

### ✔ Introduced Platform Presets

```python id="x8w1bq"
VIDEO_PLATFORM_PRESETS = {
    "youtube": (...),
    "instagram": (...),
    "reddit": (...),
    "tiktok": (...),
    "twitter": (...),
}
```

Each platform defines its domains.

---

### ✔ Enabled Platform Toggle

```python id="q4r6lm"
ENABLED_VIDEO_PLATFORMS = ("youtube", "instagram")
```

👉 Only these platforms are active.

---

### ✔ Extra Domain Support

```python id="6s2kzv"
EXTRA_VIDEO_DOMAINS = ()
```

Allows:

* custom/self-hosted sources
* quick overrides without editing presets

---

### ✔ Dynamic Domain Builder

```python id="n1p8dz"
SUPPORTED_VIDEO_DOMAINS = build_supported_video_domains()
```

* merges enabled platforms + extra domains
* removes duplicates
* normalizes values

---

### ✔ Status Visibility

```text id="c9f0yx"
/status
```

Now shows:

* enabled platforms
* supported domains
* extra domains

---

### ✔ Fail-Safe Startup Check

If no domains are enabled:

```python id="r7n3hz"
log.warning("No supported video domains are enabled")
```

---
