# Raziel YTBot v6.4

## Overview

v6.4 introduced collapsible forecast presentation using Telegram expandable blockquotes.

This version focused on reducing weather-command spam and improving forecast readability inside active chats.

Instead of dumping long multi-day weather walls into chat, Raziel now:

* keeps immediate forecast information visible
* collapses extended forecast details
* preserves full weather data on demand
* dramatically reduces vertical scroll noise

v6.4 continued the broader Raziel design direction established in v6.2 and v6.3:

* preserve useful information
* reduce unnecessary noise
* improve Telegram readability
* use expandable presentation where appropriate

---

# Core Development

## Added Collapsible Forecast System

Raziel forecasts now support Telegram expandable blockquotes for extended weather details.

Final forecast structure:

1. Current visible forecast day
2. Expandable extended forecast section

Example layout:

```text
🗓️ 5-Day Forecast

Mon — Thunderstorm
High / Low

📖 Extended Forecast
▼ expandable
```

Expanded content contains:

* remaining forecast days
* temperatures
* precipitation
* weather conditions

This significantly reduced forecast spam inside Telegram chats.

# Forecast UX Improvements

## Reduced Forecast Noise

Earlier forecast output displayed:

* full multi-day forecasts
* all forecast days at once
* large vertically stacked weather messages

v6.4 now:

* prioritizes immediate forecast visibility
* hides secondary forecast days
* preserves detail only when expanded

Result:

* cleaner scrolling
* better readability
* less interruption in active chats
* improved mobile viewing

---

## Added Configurable Forecast Expansion

New configuration options:

```python
FORECAST_COLLAPSE_DETAILS = True
FORECAST_VISIBLE_DAYS = 1
```

These allow:

* enabling/disabling collapsible forecast mode
* controlling how many forecast days remain visible before collapse

# Telegram Formatting Improvements

## Added HTML Parse Mode Handling

Telegram expandable blockquotes require HTML parsing mode.

v6.4 added:

* automatic forecast parse-mode switching
* dynamic HTML/Markdown handling
* expandable forecast compatibility fixes

This resolved issues involving:

* raw HTML tag rendering
* visible `<blockquote expandable>`
* visible `<b>` tags
* broken expandable forecast formatting

---

## Added Forecast Parse-Mode Routing

New helper:

```python
forecast_parse_mode()
```

This automatically selects:

* HTML mode for expandable forecast output
* Markdown mode for normal forecast output

This preserved compatibility with:

* existing Markdown forecast formatting
* expandable Telegram forecast rendering
* legacy weather command behavior
