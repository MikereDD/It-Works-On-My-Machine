# YTBot v5.2

## Overview
Reply-threading and group-chat noise reduction release.

This version improves how queued jobs behave in active chats by keeping command-driven downloads tied to the original request while reducing unnecessary queue confirmation noise in groups.

## Changes

### 🧵 Reply Threading Improvements

- `/dl` jobs now preserve the original command message ID
- `/audio` jobs now preserve the original command message ID
- `/clip` continues to preserve the original command message ID
- `/ui` jobs now preserve the original `/ui <url>` request after button selection
- raw shared/forwarded links continue to preserve original message context

### 🎛️ UI Callback Threading

- stored `reply_to_message_id` inside pending UI requests
- passed UI request context into queued jobs after button clicks
- final uploads now reply back to the original `/ui` request

### 🔇 Cleaner Group Behavior

- group and supergroup queue confirmations are now silent
- private chats still show full queue feedback:
  - position
  - mode
  - queue confirmation

### 📋 Queue Display Improvements

- `/queue` now shows clip ranges for clip jobs
- current clip jobs display start and end timestamps
- pending clip jobs include clip range context

## Notes

This release focuses on polish and chat flow.

The bot now behaves more naturally in groups:

- commands do not spam the chat with queue confirmations
- status messages stay attached to the original request
- final uploads remain tied to the request that created them
- `/queue` remains available when users need explicit queue state

## Impact

- cleaner group chats
- better threaded message flow
- less redundant bot output
- improved traceability from request to final upload
- more consistent behavior across `/dl`, `/audio`, `/clip`, `/ui`, and raw links
