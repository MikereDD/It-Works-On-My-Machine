# 📝 v5.3.3 — Queue Message Cleanup

> Small UX fix, big visual improvement

---

## 🔥 Problem

When links were auto-detected in group chats, the bot would send:

```text
📥 Added to queue
🔢 Position: X
🎬 Mode: video
```

But this message would remain in the chat even after the job started.

Result:

* Visual clutter in active groups
* Redundant information once processing begins
* Feels “noisy” in passive auto-watch mode

---

## 🧠 Root Cause

The queue confirmation message was:

* Sent during ingestion (`handle_url`)
* Never tracked afterward
* Never cleaned up during processing

---

## ✅ Fix

### ✔ Store queue message ID

```python
queue_msg = await message.reply_text(...)
job["queue_message_id"] = queue_msg.message_id
```

---

### ✔ Delete before upload starts

```python
queue_message_id = job.get("queue_message_id")

if queue_message_id:
    await bot.delete_message(chat_id, queue_message_id)
```

---

## 🎯 Result

* Queue message is now **temporary**
* Automatically removed when upload begins
* Chat stays clean and focused on final output

---

## 💡 Behavior Now

### Before:

```
[User message]
[Added to queue]
[Uploading...]
[Final file]
```

### After:

```
[User message]
[Uploading...]
[Final file]
```

---

## ⚠️ Notes

* Deletion is **best-effort**
* If message is already gone → safely ignored
* Requires bot to have permission to delete messages in group

---

## 🧠 Design Decision

Treat queue message as:

> **ephemeral status**, not permanent output

---

## 📌 Impact

* Improves group readability significantly
* Better UX for passive auto-watch mode
* Aligns bot behavior with “clean automation” philosophy

---

## 🚀 Summary

Small change, but important:

> The bot now cleans up after itself.
