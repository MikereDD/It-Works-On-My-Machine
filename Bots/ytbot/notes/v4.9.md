# YTBot v4.9

## Overview
Threading and reply-flow improvements for shared and forwarded links.

This version restores a cleaner group-chat experience by keeping raw shared or forwarded link downloads tied to the original message thread, making queue updates and final uploads easier to follow.

## Changes

### 🧵 Reply Threading for Shared Links

- added `reply_to_message_id` support to queued jobs
- preserved original message context for raw shared and forwarded links
- made metadata/status updates reply to the original shared message
- made final uploaded media reply to the original shared message

### 📥 Raw URL Workflow Improvements

- updated raw URL job creation to store the source message ID
- improved chat flow for users who paste or forward links directly into group chat
- restored more natural threaded behavior for passive link sharing

### 🛠️ Queue Processing Updates

- updated `process_job()` to read and use `reply_to_message_id`
- applied reply threading to:
  - metadata fetch status
  - final audio upload
  - final video upload
  - document fallback upload

### 📦 Archive Behavior

- kept archive uploads unchanged
- avoided cross-chat reply threading issues for archive destinations

## Notes

This release focuses on **message flow and readability** in active chats:

- reduces confusion in busy groups
- keeps downloads visually connected to the original shared link
- makes bot behavior feel more natural during passive use

## Impact

- better group-chat usability for shared and forwarded links
- cleaner threaded conversations
- improved traceability from original link to final uploaded media
- no change to archive workflow or command-driven downloads
