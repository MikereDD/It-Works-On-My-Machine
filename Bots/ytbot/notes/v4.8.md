# YTBot v4.8

## Overview
Role-aware help system and improved command visibility.

This version refines how users interact with the bot by tailoring `/help` and `/start`
output based on permissions and context, ensuring users only see commands relevant
to them.

## Changes

### 📖 Role-Aware Help System

- implemented dynamic `/help` output based on user role and context
- separated commands into:
  - user commands
  - admin commands
- logic now considers:
  - `is_admin(user_id)`
  - `can_use_context(user_id, chat_id, chat_type)`

### 🧭 Context-Based Visibility

- **Admins (DM or group):**
  - see full command set (user + admin)

- **Allowed group users:**
  - see user commands only

- **Unauthorized users:**
  - see minimal safe commands only

- prevents exposure of admin-only functionality in shared environments

### 🚀 Updated `/start` Command

- `/start` now mirrors role-aware behavior
- always shows user commands
- conditionally includes admin commands for authorized users

### 🧠 Command Organization

- introduced:
  - `USER_COMMAND_LIST`
  - `ADMIN_COMMAND_LIST`

- improves:
  - readability
  - maintainability
  - future extensibility of command sets

## Notes

This release focuses on **clarity and usability**:

- reduces confusion for group users
- avoids exposing irrelevant or restricted commands
- improves overall interaction flow

The role-aware approach aligns command visibility with actual permissions,
making the bot feel more intentional and polished.

## Impact

- cleaner user experience in group environments
- improved separation of admin vs user functionality
- easier onboarding for new users
- better long-term maintainability of command structure
