# Raziel v6.7 — Smarter Passive Media Detection

## Overview

Raziel v6.7 refines passive auto-watch behavior to better understand Telegram-native media sharing.

Raziel now ignores source links embedded in captions when a user has already uploaded or shared media directly into the chat.

This prevents accidental duplicate downloads caused by:

* reposted Telegram videos
* shared clips with source attribution
* forwarded media with embedded source URLs
* other bots posting media alongside original source links

---

# 🧠 Smarter Passive Watching

## Previous Behavior

Before v6.7:

```text
Telegram video uploaded
+ caption contains source URL
→ Raziel sees URL
→ queues download
```

This created unwanted duplicate downloads in active groups.

---

## New Behavior

Raziel now detects Telegram-native media first.

If a message already contains:

* video
* animation
* audio
* document
* voice
* video note

Raziel ignores caption/source links during passive auto-watch handling.

New flow:

```text
Telegram media detected
→ ignore caption URLs
→ no passive queue creation
```

# 🎯 Solved Use Cases

v6.7 prevents:

* duplicate source downloads
* repost-loop behavior
* Telegram media repost noise
* accidental ingestion from media captions
* bot-to-bot repost triggering

Examples:

```text
User uploads Telegram video
+ source YouTube link in caption
→ Raziel ignores it
```

```text
Another bot posts media with attribution link
→ Raziel ignores it
```

# ⚙️ New Internal Helper

Added:

```python
message_has_native_telegram_media()
```

This helper detects Telegram-native media payloads before passive URL ingestion begins.

# 🧹 UX Improvements

v6.7 continues Raziel’s low-noise Telegram-native design philosophy:

```text
if media already exists in chat:
    do not redundantly fetch it again
```

This creates cleaner:

* repost workflows
* source attribution behavior
* media sharing groups
* clip-sharing channels
* mixed bot ecosystems

