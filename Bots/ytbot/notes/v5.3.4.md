# 📝 v5.3.4 — Production Log Mode

> Reduce noise, keep signal

---

## 🔥 Problem

During development, the bot produced verbose logs:

```text
WATCH HIT ...
URL DETECTED ...
```

While useful for debugging, in production this caused:

* Log spam
* Reduced readability
* Harder to spot real issues

---

## 🧠 Root Cause

Debug-level logs were always enabled:

* No separation between **development** and **production**
* All messages logged at `INFO` level
* No toggle to control verbosity

---

## ✅ Fix

### ✔ Introduced DEBUG_MODE

```python
DEBUG_MODE = getattr(ytbotrc, "DEBUG_MODE", False)
```

---

### ✔ Conditional debug logging

```python
if DEBUG_MODE:
    log.info("WATCH HIT ...")

if DEBUG_MODE:
    log.info("URL DETECTED: %s", url)
```

---

### ✔ Adjusted logger level

```python
log.setLevel(logging.INFO if DEBUG_MODE else logging.WARNING)
```

---

### ✔ Added visibility in `/status`

```python
*Debug Mode:* True / False
```

---

## 🎯 Result

### Development mode (`DEBUG_MODE = True`)

* Full visibility
* Detailed message tracking
* Useful for debugging extraction and message flow

---

### Production mode (`DEBUG_MODE = False`)

* Clean logs
* Only warnings and errors shown
* Easier monitoring

---

## ⚠️ Notes

* Critical logs remain unaffected:

  * errors
  * failures
  * important state changes

* Debug logs can be re-enabled instantly via config

---

## 🧠 Design Decision

Introduce a simple toggle:

> **Verbose when needed, silent when stable**

---

## 📌 Impact

* Improves operational clarity
* Reduces log noise significantly
* Keeps debugging capability intact

---

## 🚀 Summary

This change separates:

* **building the system**
  from
* **running the system**

---

**End of v5.3.4**
