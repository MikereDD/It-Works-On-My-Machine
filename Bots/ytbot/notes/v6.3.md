# Raziel YTBot v6.3

## Overview

v6.3 refined the expandable source-context system introduced in v6.2 by adding meaningful-context detection and low-value metadata filtering.

This version focused on reducing unnecessary context noise while preserving genuinely useful creator commentary and source information.

The goal became:

* preserve meaningful context
* suppress creator-infrastructure spam
* reduce visual clutter
* keep Telegram chats readable

v6.3 evolved Raziel from:

* raw metadata preservation
  into:
* intelligent metadata presentation

---

# Core Development

## Added Meaningful Source Context Detection

Raziel now evaluates whether source metadata actually adds informational value before sending expandable context blocks.

Expandable source context is now skipped when the metadata is primarily:

* social link dumps
* “watch more here” lines
* sponsor blocks
* promo codes
* merch funnels
* repeated source URLs
* tour schedules
* timestamp/chapter spam

This dramatically reduced unnecessary secondary context messages.

---

## Added Low-Value Context Filtering

v6.3 introduced heuristic-based context cleanup designed to suppress creator-infrastructure noise while preserving meaningful commentary.

New filtering behaviors include:

* removing isolated URLs
* suppressing social platform link dumps
* filtering sponsor/promo lines
* removing repetitive creator funnel text
* filtering timestamp/chapter lines
* suppressing tour-date/event schedule formatting

The filtering logic was intentionally conservative to avoid destroying meaningful creator commentary.

---

# Expandable Context Improvements

## Cleaner Context Messages

Expandable context blocks now focus more heavily on:

* commentary
* creator statements
* contextual descriptions
* meaningful post text

while suppressing:

* redundant self-promotion
* infrastructure clutter
* low-information filler

This created:

* smaller context blocks
* less visual spam
* higher signal-to-noise ratio

---

## Preserved Non-Reply Expandable Architecture

v6.3 retained the stable v6.2 architecture:

1. Clean media upload bubble
2. Separate expandable source-context message

Context messages continue to send:

* as standalone messages
* not as replies
* using Telegram expandable blockquotes

This maintained:

* stable Telegram compatibility
* consistent rendering across clients
* clean scrolling UX

---

# Heuristic System

## Added Meaningful Context Scoring

v6.3 introduced heuristics to determine whether metadata is worth expanding.

Checks include:

* minimum meaningful character count
* minimum unique-word density
* URL-heavy content suppression
* duplicate-title detection
* low-information phrase detection

This allows Raziel to suppress context that adds little or no informational value.

---

## Added Noise Pattern Detection

v6.3 added detection patterns for:

* “watch more”
* “subscribe”
* “promo code”
* “follow me”
* “tour dates”
* “premium content”
* platform/social dumps

The system now suppresses many common creator-marketing structures automatically.
