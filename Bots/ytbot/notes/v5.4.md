# 📝 v5.4 — Dedupe + Smarter Downloads

> Stop wasting cycles. Start making better choices.

---

## 🔥 Problem

Two major inefficiencies existed in the pipeline:

### 1. Duplicate Downloads

* Same link shared multiple times in groups
* Forwarded messages re-triggered downloads
* Tracking params (`?si=`, `&feature=`) bypassed equality checks

Result:

* Wasted bandwidth
* Duplicate files
* Queue pollution

---

### 2. Suboptimal Format Selection

* yt-dlp sometimes chose:

  * non-MP4 containers
  * unnecessarily large formats
  * incompatible streams

Result:

* Larger downloads than needed
* More compression work
* Slower pipeline

---

## 🧠 Solution

### ✔ Persistent Dedupe System

#### Normalize URLs

```python
def normalize_url_for_dedupe(url):
```

Handles:

* `youtu.be → youtube.com/watch?v=`
* mobile → standard domain
* strips tracking/query noise

---

#### Cache with TTL

```python
DEDUP_FILE = state/dedup.json
DEDUP_TTL_HOURS = 24
```

Each entry stores:

* url
* job_id
* chat
* timestamp

---

#### Duplicate detection

```python
if is_duplicate_url(url):
    return
```

👉 Happens **before queueing**

---

#### Auto cleanup

```python
prune_dedup_cache()
```

* Removes expired entries
* Keeps cache small and relevant

---

### ✔ Smarter yt-dlp Formats

Replaced:

```text
bv*+ba/b
```

With:

```text
bv*[height<=MAX][ext=mp4]+ba[ext=m4a]
```

Fallback chain:

```text
mp4 → mixed → best
```

---

### ✔ Configurable Behavior

```python
DEDUP_ENABLED = True
DEDUP_TTL_HOURS = 24

MAX_VIDEO_HEIGHT = 1080
PREFER_MP4 = True
```

---

### ✔ Status Visibility

```text
/status
```

Now shows:

* dedupe enabled
* TTL window
* max resolution
* MP4 preference
