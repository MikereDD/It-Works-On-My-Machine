# 📝 v5.4.3 — Quality Control Commands

> Give users control over download quality.

---

## 🔥 Problem

Previously, video downloads were controlled by a **single global setting**:

```text
MAX_VIDEO_HEIGHT
```

This caused:

* No per-request flexibility
* Users forced into one quality level
* Inefficient downloads (too large or too small)

---

## 🧠 Root Cause

Quality handling was:

* global
* static
* not user-controlled

---

## ✅ Fix

### ✔ Introduced Per-Job Quality

Each job now includes:

```python
quality = "default" | "hd" | "full"
```

---

### ✔ New Commands

| Command | Behavior                |
| ------- | ----------------------- |
| `/dl`   | 720p (default)          |
| `/hd`   | 1080p                   |
| `/full` | Best available (no cap) |

---

### ✔ UI Improvements

Inline UI now supports:

* 🎬 720p
* 🎬 HD
* 🎬 Full
* 🎵 Audio

---

### ✔ Dynamic yt-dlp Selection

```text
default → height ≤ 720  
hd      → height ≤ 1080  
full    → no height restriction  
```

---

### ✔ Config Support

```python
DEFAULT_VIDEO_HEIGHT = 720
HD_VIDEO_HEIGHT = 1080
```

---

### ✔ Queue Visibility

```text
/queue
```

Now shows:

* mode
* quality

---

### ✔ Status Improvements

```text
/status
```

Now displays:

* default quality
* HD quality
* legacy max height (fallback)

---

## 🎯 Result

### Before

* One global quality setting
* No user control

---

### After

* Per-request quality selection
* Better bandwidth usage
* Cleaner user experience

---

## ⚠️ Notes

* `/full` may produce very large files
* Telegram upload limits still apply (~2GB local API)
* `/audio` unaffected

---

## 🧠 Design Philosophy

> Let users choose
> without breaking the pipeline

---

## 📌 Impact

* Improved usability
* Smarter downloads
* More flexible pipeline

---

## 🚀 Summary

This version upgrades the bot from:

**"fixed quality downloads"**

to

**"user-controlled quality system"**
