# YTBot v5.0

## Overview
Clip support and media workflow expansion.

This version introduces `/clip` as a first-class feature, allowing users to trim a section from a video using start and end times while preserving the same upload experience used by the rest of the bot.

## Changes

### ✂️ Clip Support

- added `/clip <url> <start> <end>`
- supports clip time input in:
  - `MM:SS`
  - `HH:MM:SS`

- validates:
  - time format
  - start/end ordering
  - ffmpeg availability

### 🎞️ Clip Processing Pipeline

- introduced clip-aware jobs in the queue system
- extended job model with:
  - `clip_start`
  - `clip_end`

- added video clipping stage using `ffmpeg`
- clip jobs download source video, trim it, then upload the result

### 🏷️ Metadata Retention

- clip uploads retain metadata captions, including:
  - title
  - uploader
  - source link
  - clip range

- caption formatting remains consistent with `/ui`, `/dl`, and `/audio`

### 📤 Upload & Routing Behavior

- clip output follows normal video upload flow
- supports:
  - Telegram upload fallback behavior
  - archive sending
  - routing into final video output storage

- clip uploads continue to respect reply threading where applicable

### 🧠 Command & UX Updates

- added `/clip` to user-facing help and command listings
- promoted clip creation to a primary bot capability
- expanded the bot from download-only workflows into lightweight media editing

## Notes

This release marks a meaningful step forward in the bot’s scope:

- previous versions improved control, usability, and presentation
- v5.0 introduces a new media-processing capability
- the bot now supports both downloading and transforming media

This makes v5.0 a natural milestone release.

## Impact

- users can extract only the part of a video they want
- metadata and attribution remain intact
- queue architecture now supports richer media job types
- bot utility expands beyond download and upload into practical media editing
