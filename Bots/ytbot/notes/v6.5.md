# Raziel v6.5 — Reply-Driven Download Commands

## Overview

Raziel v6.5 introduces reply-driven media commands for cleaner and more natural Telegram workflows.

Instead of reposting URLs or pasting links repeatedly, users can now reply directly to a message containing a supported media link and trigger Raziel actions using short reply commands.

This creates a low-noise, Telegram-native workflow for group chats and shared media channels.

---

# New Features

## Reply-Driven Commands

Reply to a message containing a supported media URL with:

```text
/rdl
/rhd
/rfull
/raudio
/rui
```

Raziel automatically extracts the URL from the replied-to message and processes it.

---

# Commands

## /rdl

Reply-download using the default video profile.

Default target:

* 720p optimized output

---

## /rhd

Reply-download HD video.

Default target:

* 1080p preferred output

---

## /rfull

Reply-download the best/full available quality.

Attempts:

* highest available resolution
* best source quality
* largest compatible media output

---

## /raudio

Reply-download audio extraction.

Used for:

* podcasts
* music
* interviews
* spoken content
* audio-only workflows

---

## /rui

Reply-open interactive quality selection UI.

Provides:

* format selection
* quality preview
* interactive download buttons

---

# Workflow Example

```text
User posts:
https://youtube.com/watch?v=example

Reply:
/rdl
```

Raziel:

```text
→ extracts replied-to URL
→ queues media
→ processes download
→ replies with result
```

---

# UX Improvements

## Reduced Chat Noise

Reply command messages are automatically deleted after successful queueing.

This keeps chats focused on:

* media
* queue status
* completed downloads

instead of command clutter.

# Internal Improvements

* Added reply-message URL extraction helpers
* Added reply-driven command routing
* Added automatic command cleanup
* Integrated reply commands into queue workflow
* Preserved deduplication system support
* Preserved threaded reply behavior
* Preserved UI workflow support

---

# Notes

Reply commands only work when:

* replying to a message
* the replied-to message contains a supported media URL

Unsupported or invalid URLs are rejected safely.
