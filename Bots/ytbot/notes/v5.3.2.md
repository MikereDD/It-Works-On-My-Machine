## 📌 v5.3.2 — Group Auto-Watch Fix (MAJOR)

### 🔥 Problem

Bot was:

* Working with `/dl`, `/audio`, `/clip`
* NOT reacting to shared / forwarded links in group chats

---

### 🧠 Root Causes

1. **Message filtering issue**

```python
if not message or not message.text:
    return
```

* Blocked:

  * shared links (caption-based)
  * forwarded links (entity-based)

---

2. **Handler scope too narrow**

```python
filters.TEXT | filters.Caption()
```

* Missed:

  * non-text message types
  * forwarded previews

---

3. **Permission gating blocked execution**

```python
if not can_use_context(...):
    return
```

* Required `chat_id` in `ALLOWED_CHAT_IDS`
* Silent failure if mismatch

---

4. **Wrong group ID in config**

Expected:

```text
-1003955245414
```

But config used:

```text
-1003344125467
```

---

## ✅ Fixes Applied

### ✔ Removed message.text restriction

```python
if not message:
    return
```

---

### ✔ Expanded handler scope

```python
filters.ALL & ~filters.COMMAND
```

---

### ✔ Rebuilt URL extraction logic

Supports:

* message.text
* message.caption
* entities (url + text_link)
* raw payload fallback

---

### ✔ Updated permission model

```python
if chat_type in ("group", "supergroup"):
    return True
```

👉 All groups now supported automatically

---

### ✔ Added debug logging

```python
WATCH HIT
URL DETECTED
```

Used to confirm:

* message delivery
* extraction success
