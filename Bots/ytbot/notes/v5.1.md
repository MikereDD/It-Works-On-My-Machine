# Raziel YTBot — v5.1

## Overview
v5.1 restores the bot to a stable working state after rolling back experimental changes that introduced instability. It combines the proven v5.0 logic with improved path handling and corrected upload limits.

---

## Changes

### 🔁 Rollback to Stable Core
- Restored last known stable logic from v5.0
- `/dl`, `/audio`, and `/clip` confirmed working
- Removed unstable experimental features (compression/splitting changes)

---

### 📁 Path System Upgrade (from v5.2.1)
- Replaced hardcoded paths with dynamic resolution

```python
APP_DIR = Path(__file__).resolve().parent
ROOT_DIR = APP_DIR.parent

CONFIG_DIR = ROOT_DIR / "config"
CONFIG_FILE = CONFIG_DIR / "ytbotrc.py"

_configured_base = getattr(ytbotrc, "BASE_DIR", ROOT_DIR)
BASE_DIR = Path(os.path.expandvars(os.path.expanduser(str(_configured_base)))).resolve()
```

#### Benefits:
- Cross-platform support (Windows + Linux)
- No more hardcoded `G:/bots` dependency
- Cleaner project structure

---

### 📦 Upload Limit Fix
- Updated Telegram upload limit handling

**Before:**
```python
MAX_UPLOAD_BYTES = 49 * 1024 * 1024
```

**After:**
```python
MAX_UPLOAD_BYTES = 1900 * 1024 * 1024
```

- Prevents unnecessary compression for normal files
- Files under ~2GB upload directly

---

### 🧹 Removed Behavior
- Disabled automatic compression for typical files
- Removed broken large-file handling logic introduced after v5.0

---

## Current Behavior

| Scenario | Result |
|----------|--------|
| Small file (< 2GB) | Direct upload |
| Large file (> 2GB) | Not yet handled (future feature) |
| `/clip` | Fully working |
| `/audio` | Fully working |

---

## Known Limitations
- No fallback splitting for >2GB files (planned for v5.3)
- Download timeout still default (may require tuning)
- No progress reporting during download

---

## Next Steps (v5.2 → v5.3)

- Add upload progress indicators
