# Raziel v6.6 — Media-Aware Queue Validation & Cleanup Controls

## Overview

Raziel v6.6 introduces media-aware preflight validation before queueing downloads.

Raziel no longer treats every supported URL as valid downloadable media.

Instead, Raziel now verifies that a link actually exposes downloadable video or audio before:

* queueing
* processing
* generating failure noise

This significantly improves Telegram UX by preventing unnecessary:

* queue spam
* failed downloads
* noisy error chains
* useless processing attempts

especially from:

* text-only X/Twitter posts
* Reddit self posts
* non-media social links
* unsupported embedded content

---

# 🧠 Media-Aware Queue Validation

## Previous Behavior

Before v6.6:

```text
Supported URL
→ queue
→ attempt download
→ fail later
```

This produced unnecessary noise for:

* text-only tweets
* article links
* non-video social posts

---

## New Behavior

Raziel now performs a lightweight yt-dlp metadata preflight before queueing.

New flow:

```text
URL detected
→ media preflight
→ downloadable media exists?
→ YES → queue
→ NO → ignore/temporary notice
```

This evolves Raziel from:

```text
URL-aware
```

into:

```text
media-aware
```

---

# 🔍 Media Preflight System

New internal helpers:

```python
info_has_downloadable_media()
media_preflight_check()
media_preflight_allows_queue()
```

The system now evaluates:

* formats
* codecs
* direct media URLs
* requested downloads
* extractor metadata

before allowing queue creation.

---

# 🌐 Supported No-Media Detection

v6.6 now suppresses noisy queue/failure behavior for:

* text-only X/Twitter posts
* Reddit self posts
* non-media social links
* unsupported extractor payloads
* empty media containers

---

# 🧹 Cleanup & Noise Reduction

## Temporary Error Cleanup

Predictable low-value error messages now auto-delete after a short delay.

Examples:

* no supported URL
* unsupported media
* duplicate queue attempt
* no downloadable media found

New config:

```python
NOISE_MESSAGE_DELETE_SECONDS = 12
MEDIA_PREFLIGHT_NOISE_DELETE_SECONDS = 8
```

---

# 🗑️ Manual Cleanup Commands

Raziel now supports manual cleanup controls.

Reply to a Raziel message with:

```text
/delete
/del
/rm
```

Raziel will:

* delete the replied-to bot message
* delete the cleanup command itself
* silently ignore Telegram deletion failures

This creates a significantly cleaner moderation and operational workflow for active groups.
