# YTBot v4.7

## Overview
Metadata enrichment for uploaded media with source-aware behavior.

This version improves how downloaded content is presented by attaching
relevant metadata (title, uploader, and source link) directly to uploaded files,
while preserving clean behavior for automated and passive workflows.

## Changes

### 🎞️ Metadata Captions

- added captions to uploaded media (video, audio, document fallback)
- captions include:
  - title
  - uploader (channel/author)
  - original source link

- applied to:
  - `/ui`
  - `/dl`
  - `/audio`

### 🧠 Source-Aware Behavior

- introduced `source` tagging for jobs:
  - `ui`, `dl`, `audio` → captions enabled
  - `raw_url`, `watch` → captions disabled

- ensures:
  - manual user actions get rich metadata
  - automated or passive workflows remain clean and minimal

### 📦 Upload Consistency

- applied caption behavior to:
  - standard uploads
  - archive uploads (`ARCHIVE_CHAT_ID`)

- ensured fallback uploads (`send_document`) also include captions

### 🔧 Queue & Job Handling

- updated job creation in:
  - `/dl`
  - `/audio`
  - raw URL handler

- ensured proper queue placement and source tagging consistency

## Notes

This release focuses on **presentation and usability**:

- makes shared media self-describing
- improves traceability of downloaded content
- avoids clutter in automated workflows

The source-aware design ensures flexibility without introducing
additional configuration complexity.

## Impact

- better user experience when sharing downloaded media
- clearer attribution of content origin
- no disruption to existing automation or watch-folder behavior
- maintains clean output for passive URL ingestion
