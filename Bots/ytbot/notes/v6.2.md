# Raziel YTBot v6.2

## Overview

v6.2 introduced expandable source-context support for downloaded media posts.

This version focused on preserving creator/source context while keeping media uploads visually clean and readable inside Telegram chats.

The original goal was to place expandable source context directly inside Telegram media captions. Through testing, it was discovered that Telegram handles expandable blockquotes reliably in normal messages, but not consistently in media captions across clients.

v6.2 ultimately stabilized around:

* clean media captions
* separate expandable context messages
* non-reply context delivery
* preserved source metadata without destructive filtering

This became the architectural foundation for future context refinement improvements later expanded in v6.3.

---

# Core Development

## Added Expandable Source Context

Raziel can now preserve source metadata using Telegram expandable blockquotes.

Supported metadata sources include:

* YouTube descriptions
* X/Twitter post text
* yt-dlp post metadata
* creator commentary
* attached source descriptions

Expandable context allows users to:

* expand long source descriptions only when desired
* reduce visible chat clutter
* preserve archival/source information

---

## Separated Media Caption From Context

v6.2 intentionally separated:

* media upload captions
* expandable source context

Final upload flow:

1. Clean media upload bubble
2. Separate expandable context message

This avoided:

* caption parser instability
* malformed HTML entities
* Telegram media-caption limitations
* inconsistent client rendering

---

## Removed Reply-Thread Context Behavior

Earlier v6.2 testing used:

* reply-to-upload context messages

This produced:

* “Raziel replied to Raziel”
* visual fragmentation
* stacked message clutter

The final v6.2 direction sends expandable context as:

* standalone non-reply messages

Result:

* cleaner chat flow
* cleaner scrolling behavior
* more intentional UX

---

# Caption System Improvements

## Cleaner Upload Captions

Media captions were simplified to:

* title
* uploader
* platform
* duration
* source URL

This created:

* consistent visual formatting
* smaller upload bubbles
* improved readability
* less metadata noise

---

## Preserved Full Source Metadata

Instead of aggressively filtering descriptions, v6.2 preserved:

* creator commentary
* contextual text
* tweet content
* relevant post descriptions

This avoided destructive cleanup logic and preserved source integrity.

# Telegram Formatting Research

## Expandable Blockquote Testing

v6.2 explored Telegram expandable quote support extensively.

Findings:

* expandable blockquotes work reliably in normal messages
* expandable blockquotes do NOT reliably work in media captions
* HTML parsing inside captions is inconsistent across Telegram clients
* raw HTML tags may appear when media captions reject entity parsing

This research directly influenced the final stable v6.2 architecture.
