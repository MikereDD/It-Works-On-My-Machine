# YTBot v4.0

## Overview
Introduction of the queue-based core system.

## Changes
- replaced direct execution model with queue-based processing
- added persistent queue storage (queue.json)
- added history tracking (history.json)
- added failure tracking (failures.json)
- implemented single worker loop to process jobs sequentially
- introduced job structure (id, user, chat, mode, timestamp)
- added access control system (owner, allowed users)
- added core commands:
  - /queue
  - /clearqueue
  - /failures
  - /retrylast
  - /stats

## Notes
This version establishes the foundation for all future features by separating input handling from processing.

## Impact
Major architectural shift. Enables stability, scalability, and future feature expansion.
