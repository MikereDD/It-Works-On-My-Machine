# Raziel / YTBot v5.9.1

## Overview

v5.9.1 focused on Telegram UX cleanup, inline identity consistency, and improved direct-message command behavior. This release refined how mention commands behave across private chats, groups, and inline mode while eliminating confusing fake clickable aliases.

---

## Telegram Mention & Inline Improvements

### Real Username Standardization

* Standardized user-facing examples around the real bot identity:

  * `@Razi3l_bot`
* Removed misleading `@Raziel` clickable examples from help output and parser documentation.

### Inline Mode Cleanup

* Clarified distinction between:

  * Telegram inline mode
  * in-chat mention commands
* Updated help output to correctly describe:

  * inline/mention weather
  * inline/mention forecast

### Help Formatting Fixes

* Wrapped inline examples in backticks to preserve:

  * underscores
  * clickable formatting
  * username rendering

Example:

```text
`@Razi3l_bot weather Houston`
```

This prevents Telegram from visually collapsing:

```text
@Razi3l_bot
```

into malformed display output.

---

## Direct Message Command Improvements

### Natural Commands in DMs

Raziel now supports natural commands in private chats without requiring mentions.

Examples:

```text
weather Houston
forecast Tokyo
queue
status
help
```

This creates cleaner direct-message UX while preserving stricter mention behavior in groups.

### Group Safety

Group chats still require:

```text
@Razi3l_bot weather Houston
```

or slash commands to avoid accidental command triggering in conversations.

---

## Mention Parser Cleanup

### Alias Handling Improvements

* Cosmetic aliases remain supported for joined chats.
* Fake clickable aliases are no longer advertised in help output.
* Parser behavior is now aligned with actual Telegram limitations.

### Improved Parser Documentation

Updated parser examples and internal comments to reflect:

* actual Telegram behavior
* inline restrictions
* proper mention usage

---

## Stability & UX Refinement

v5.9.1 primarily focused on:

* operational polish
* parser consistency
* Telegram UX correctness
* safer help formatting
* reducing user confusion around inline mentions

This release finalizes the mention/inline cleanup work introduced during the v5.8.x → v5.9 transition.
